Getting Started

Internationalize your enterprise application in the most simplified and elegant manner.

How to start?

You can internationalize your application in just two simple steps without much additional endeavour. To start with that, you need to install @rxweb/translate package.

npm install @rxweb/translate

You can either automatically set up all the necessary configuration needed for the translation and you can check every detail by configuring manually in your application. Let's have a look at how it is done in both ways.

Manual Setup for Translation

For manual setup, here are those 2 simple steps which I discussed above:

  1. Once the above package(@rxweb/translate) is installed, you need to register the RxTranslateModule in the main module of the application.
  2. Map @translate decorator on the respective component property.

Have a look at the detailed steps now.

Register The Module

Import RxTranslateModule with the resolution path of the translation content JSON file.

Map Translate Decorator

Next step is to map translate decorator on that component property where you want to display the translated value. Here it is done with GlobalComponent. If you want to display with any other translation then you can pass it's name in translationName property
Now your application is ready for the internationalization. You just need the JSON file for your translation content in their respective folders. Default location of those JSON files is assets/i18n/[languagecode]. Here shown a sample JSON file.

Automatically Configure Translation With CLI

You can automatically configure everything which is needed to set the translation in your application through Rxweb CLI. Seems much easy right? ðŸ˜‰ You just need to install @rxweb/cli globally in your system and you'll be able to configure everything automatically.

npm install -g @rxweb/cli

Once the CLI is installed globally in your system, open command prompt at the project directory and run rxweb command.

  1. Go to your project directory.
  2. Open command prompt at that location.
  3. Run rxweb command.

After running the above rxweb command in your application, below files/folders will be generated:

  • src/app/rxweb.module.ts: This file resolves RxTranslateModule which contains caching and filePaths. You need to import this RxWebModule in your root module.
  • rxweb.json: This file contains internationalization related configuration.
  • rxweb.webpack.dev.js: This file webpack configuration file which exports TranslationWebpackPlugin module.
  • assets/i18n: This folder contains multilingual data in a JSON format in their respective language folders
  • assets/i18n/en/global.en.json: This global multilingual data file used as default data file if you don't mention any translation name.
  • src/app/i18n-models: This folder contains models for strongly typed binding.

This will automatically resolve the RxTranslateModule and set the configuration needed for internationalization. Same as the manual process you need to map translate decorator on that component property where you want to display the translated value.
